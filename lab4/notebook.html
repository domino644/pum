Ryzyko kredytowe

Kada instytucja udzielajca kredytu musi szacowa zdolno kredytow i prawdopodobiestwo, czy dana osoba spaci kredyt. W praktyce wikszo osb paci prawidowo, wic zamiast tego przewiduje si prawdopodobiestwo, e kto bdzie mia opnienia w patnoci w okrelonym czasie, tzw. probability of default. Jak sama nazwa wskazuje, nie jest to klasyfikacja binarna, tylko faktyczne prawdopodobiestwo, wykorzystywane przy ocenie ryzyka.

Wykorzystamy zbir z konkursu Give Me Some Credit. Jego zmienne to (rdo):

  • SeriousDlqin2yrs - brak spaty raty kredytu przez 90 dni lub duej w cigu ostatnich 2 lat; s to nasze klasy y
  • RevolvingUtilizationOfUnsecuredLines - aktualne zapoyczenie na kartach kredytowych i poyczkach w stosunku do maksymalnego moliwego zapoyczenia
  • age - wiek w latach
  • DebtRatio - aktualne cakowite zapoyczenie wzgldem miesicznego dochodu
  • MonthlyIncome - miesiczne dochody
  • NumberOfOpenCreditLinesAndLoans - liczba aktualnych poyczek i kart kredytowych
  • NumberOfTimes90DaysLate - ile razy w przeszoci wystpio opnienie 90 dni lub duej w spacie raty kredytu
  • NumberRealEstateLoansOrLines - liczba poyczek z nieruchomoci jako zabezpieczeniem
  • NumberOfDependents - liczba pozostaych osb w gospodarstwie domowym
  • NumberOfTime30-59DaysPastDueNotWorse - ile razy w cigu ostatnich 2 lat nastpio opnienie w spacie raty midzy 30 a 59 dni
  • NumberOfTime60-89DaysPastDueNotWorse - jak wyej, ale 60-89 dni

Ze wzgldu na potrzeb precyzyjnych prawdopodobiestw oraz rozmiar danych (banki maj ogromn liczb klientw), regresja logistyczna to idealny wybr dla tego zadania. Metryk w takich przypadkach jest AUROC (Area Under Receiver Operating Characteristic curve), bo bierze ona pod uwag prawdopodobiestwo klasy pozytywnej.

Na pocztek wczytamy i przeanalizujemy wstpnie nasze dane. Jako zbir testowy wykorzystamy 20% danych. Jako e mamy do czynienia z klasyfikacj niezbalansowan, to wykorzystamy podzia ze stratyfikacj (stratified split), w ktrym proporcje klasy pozytywnej i negatywnej zostaj zachowane w zbiorze treningowym i testowym.

Detekcja spamu

Wykorzystamy teraz klasyfikator Naiwnego Bayesa do klasyfikacji binarnej tekstw - czy email jest spamem, czy te nie (tzw. "ham"). Takie zadanie nazywa si te spam/ham classification. Zbiorem bdzie Enron Spam dataset, zebrany na potrzeby artykuu:

Metsis, Vangelis, Ion Androutsopoulos, and Georgios Paliouras. "Spam filtering with Naive Bayes - which Naive Bayes?." CEAS. Vol. 17. 2006. link

Upadek firmy energetycznej Enron i skandal wywoany wiadomociami o nieprawidowociach finansowych spowodoway, e podczas ledztwa i procesu zmuszono firm do upublicznienia penej zawartoci skrzynek mailowych 150 pracownikw. Jest to doskonae rdo prawdziwych danych, typowych emaili niebdcych spamem, i byo wykorzystywane wielokrotnie na potrzeby tworzenia wielu rnych zbiorw danych. Na potrzeby powyszego artykuu wybrano 6 pracownikw z du liczb emaili.

Maile bdce spamem otrzymano z 4 rnych rde:

  1. Zbioru danych SpamAssassin - stworzony z pomoc Apache SpamAssassin, narzdzia open source do detekcji spamu. Maile niebdce spamem s w nim bardzo specyficzne (pochodz ze skrzynek fundacji Apache i dotycz praktycznie tylko programownaia), za to spam jest uniwersalny.
  2. Project Honeypot - projekt majcy na celu wykrywanie spammerw (w tym przez specjalne puapki, tzw. spam baiting) i tworzenie baz do filtrowania spamu.
  3. Baz spamu Bruce'a Guentera - gromadzi on cay spam, ktry dostaje na skrzynki (w tym takie do spam baitingu) od 1997 roku.
  4. Maile ze spamem zgromadzone przez trzeciego autora artykuu, Georgiosa Paliourasa.

Artyku prezentuje spersonalizowane kolekcje maili, dla kadego z 6 wybranych pracownikw Enronu. My natomiast poczymy wszystkie dane i stworzymy prosty, uniwersalny filtr. Nasz filtr bdzie operowa tylko na tytuach i treci maili - realnie wdraane systemy s duo bardziej rozbudowane i wykorzystuj metadane, bazy adresw, analiz tagw HTML i XML etc.

Ten zbir w caoci jest o tyle specyficzny, e dodano tutaj a tyle spamu, e jest go nawet wicej ni zwykych emaili. W praktyce spamu jest duo mniej, ok. 1/3 maili. Tutaj uznajemy, e klasy s zbalansowane.

Typow metryk w klasyfikacji spamu jest precyzja (precision), bo wyznacza ona, jak dobrzy jestemy w oznaczaniu spamu. Recall ma mniejsze znaczenie, bo bardziej zaley nam na minimalizacji FP - wane maile nie mog trafia do spamu, natomiast jeeli spam czasem trafi do zwykej skrzynki (FN), to nie ma wielkiego znaczenia. Czsto uywa si te F1-score jako metryki.

W przypadku tych metryk (precision/recall/F1-score) zawsze trzeba sprawdzi co najmniej dwie z nich, eby unikn przypadkiem sytuacji, e mamy np. precision bliskie 100%, a dramatycznie niski recall - to te nie byoby akceptowalne. from sklearn.base import BaseEstimator, ClassifierMixin
from sklearn.utils import check_array, check_X_y
import numpy as np

class CustomGaussianNB(BaseEstimator, ClassifierMixin):
    def __init__(self, var_smoothing=1e-9):
        self.var_smoothing = var_smoothing

    def fit(self, X, y):
        # Validate input
        X, y = check_X_y(X, y)
        
        # Get unique classes
        self.classes_ = np.unique(y)
        n_classes = len(self.classes_)
        n_features = X.shape[1]

        # Initialize parameters
        self.log_priors_ = np.zeros(n_classes)
        self.means_ = np.zeros((n_classes, n_features))
        self.variances_ = np.zeros((n_classes, n_features))

        # Calculate parameters for each class
        for idx, cls in enumerate(self.classes_):
            X_cls = X[y == cls]
            self.log_priors_[idx] = np.log(X_cls.shape[0] / X.shape[0])
            self.means_[idx] = X_cls.mean(axis=0)
            self.variances_[idx] = X_cls.var(axis=0) + self.var_smoothing

        return self

    def predict(self, X):
        # Validate input
        X = check_array(X)

        # Predict class for each sample
        return np.array([self._single_sample_predict(x) for x in X])

    def _single_sample_predict(self, x):
        # Calculate log-posterior for each class
        log_posteriors = []
        for idx, _ in enumerate(self.classes_):
            log_prior = self.log_priors_[idx]
            log_likelihood = -0.5 * np.sum(
                np.log(2 * np.pi * self.variances_[idx])
                + ((x - self.means_[idx]) ** 2) / self.variances_[idx]
            )
            log_posteriors.append(log_prior + log_likelihood)

        # Return class with maximum log-posterior
        return self.classes_[np.argmax(log_posteriors)]
